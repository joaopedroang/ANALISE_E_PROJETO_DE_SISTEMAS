Lista de Requisitos: Cenário Conta de Luz
1. Requisitos Funcionais (RF)
Estes requisitos descrevem as funcionalidades diretas solicitadas no cenário:

RF01 - Cadastrar Conta de Luz: O sistema deve permitir a inserção de dados das contas (data de leitura, número da leitura, kW gasto, valor a pagar, data de pagamento e média de consumo).

RF02 - Listar Acompanhamento: O sistema deve exibir uma lista com todos os gastos mensais registrados.

RF03 - Verificar Menor Consumo: O sistema deve identificar e destacar o mês que apresentou a menor quantidade de kW gasto.

RF04 - Verificar Maior Consumo: O sistema deve identificar e destacar o mês que apresentou a maior quantidade de kW gasto.

RF05 - Calcular Média de Consumo: O sistema deve realizar automaticamente o cálculo da média de consumo diário para cada registro inserido.

2. Requisitos Não Funcionais (RNF)
Estes requisitos referem-se a restrições técnicas ou qualidades do sistema:

RNF01 - Armazenamento: Os dados devem ser persistidos (salvos) para que as consultas mensais sejam realizadas sobre um histórico.

RNF02 - Usabilidade: A interface deve seguir o modelo de visualização em tabela (similar à planilha Excel apresentada no cenário).

RNF03 - Precisão: Os campos de "Valor a Pagar" e "Média de Consumo" devem suportar precisão de duas casas decimais (tipo double ou decimal).

RNF04 - Disponibilidade: O sistema deve estar disponível para consultas mensais rápidas por parte do usuário (Gabriel).
