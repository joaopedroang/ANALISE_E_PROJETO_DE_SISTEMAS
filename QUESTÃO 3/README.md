# Exercício 02 - Classe Boneco em Movimento
## Cenário
O projeto consiste no desenvolvimento de uma classe para controlar a movimentação de um boneco em um plano cartesiano (X, Y). O sistema permite definir o nome do boneco, controlar sua direção (cima, baixo, esquerda, direita) e atualizar sua posição conforme os comandos do usuário, sob a orientação do Prof. Ricardo Roberto.

## 1. Identificação de Classes, Atributos e Métodos
Classe: Boneco

Atributos: nome, posicaoX, posicaoY, direcaoAtual.
Métodos: mudarDirecao(novaDirecao), mover(), exibirStatus().

## 2. Requisitos Funcionais
[RF01] O sistema deve permitir criar um boneco com nome e posição inicial.
[RF02] O boneco deve possuir uma direção atual.
[RF03] O sistema deve permitir alterar a direção do boneco.
[RF04] O boneco deve se mover para cima, baixo, esquerda e direita.
[RF05] O sistema deve atualizar a posição do boneco após cada movimento.
[RF06] O sistema deve exibir a posição atual do boneco.
[RF07] O sistema deve exibir nome, posição e direção.

## 3. Requisitos Não Funcionais
[RNF01] O sistema deve ser simples e de fácil entendimento.
[RNF02] O código deve ser organizado seguindo boas práticas de POO.
[RNF03] O sistema deve ter boa legibilidade (nomes claros).
[RNF04] O tempo de resposta para movimentação deve ser o menor possível.
[RNF05] O sistema deve permitir fácil manutenção e expansão.
